**Chronolith gives a repository a verifiable memory.** One hash covers every
tracked file, keyed by its path, so a one-byte edit — or a rename, or two files
swapping contents — moves the root. A signed baseline turns *"did this change?"*
from an opinion into a question with a cryptographic answer.

```bash
pip install chronolith-lite
chronolith-lite init
chronolith-lite check
```

Now edit any tracked file and run `check` again. Watch the root move. That is
the whole idea, and it takes a minute.

### Lite — `pip install chronolith-lite`

Drift detection with no ceremony. Four commands: `init`, `check`, `status`,
`log`. It crystallizes a Merkle root over your documentation and blocks a push
when the tree stops matching the baseline.

Start here. Pro is a superset, so nothing you learn is wasted.

### Pro — `pip install chronolith-pro`

Everything in Lite, plus proof a stranger can check without trusting you.

```bash
chronolith-pro sovereign-init     # Ed25519 signing + X25519 encryption keys
chronolith-pro check              # the baseline is now really signed
chronolith-pro verify             # read-only; the command a skeptic runs
```

`verify` recomputes the root, checks the Ed25519 signature and the append-only
transparency chain, and — the part worth reading — **states what it did not
prove**:

```
! authenticity NOT verified: no --expect-fingerprint. This proves internal
  integrity only; a fork that swapped the whole key set would still pass.
```

Also included: Merkle inclusion proofs (`prove`) that verify one file belongs to
the root in O(log n) hashes without rehashing the repository, provenance
attestations (`attest`), key rotation that keeps history signed by retired keys
verifiable, and Bitcoin anchoring through OpenTimestamps. A real anchor is
committed in [`docs/evidence/`](https://github.com/SteveBlackbeard/CHRONOLITH-by-Ethernium/tree/main/docs/evidence)
— block 958484.

### Omega — `pip install chronolith-omega`

Lite and Pro answer *"has this changed?"*. Omega answers *"why did we decide
this?"* — semantic search over the governed corpus, entirely local.

```bash
chronolith-omega init
chronolith-omega index
chronolith-omega query "why did we choose Ed25519 for signatures?"
```

Retrieval is by meaning rather than keywords, and it crosses languages: an
English question finds a Spanish decision log that shares none of its words.

## What this release fixes

3.2.1 exists because we ran the software instead of reading it. Two things had
**never worked at all**:

- **`chain`** raised `NameError` on every invocation since the rename. It
  rendered the transparency chain using rich's `Table` without importing it — a
  documented, shipped command that had never once succeeded, because no test
  ever ran it.
- **The Sentinel CI job** had never succeeded either. `git add` aborts across
  all of its arguments when one path is missing, so nothing was ever staged,
  and the drift check then read the working tree instead of the index.

Also fixed: the Guardian and the Sentinel were reporting each other's writes as
drift, which kept the build red through nobody's fault; the golden baseline had
been stale since the rename; the secret scanner could not see private keys,
because the extension filter skipped `.priv` before ever reading it; and
root-level `pytest` reported 34 false failures caused by the three editions
shadowing each other's package.

There is now a test that discovers every command from the CLI and runs the
read-only ones against a real project. A command may refuse. It may not crash.

**First release published through Trusted Publishing.** No API token exists in
the workflow; PyPI's attestations record which repository and workflow built
each file.
