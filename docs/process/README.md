# Process Documents

This folder contains operational documents used to prepare releases, track sessions, and verify publication readiness.

These files are important, but they are not first-screen product documentation. Keeping them here reduces root noise while preserving the release trail.

## Contents

- `REPO_AND_PYPI_RELEASE_CHECKLIST.md`: GitHub and PyPI release checklist.
- `RELEASE_STAGING_PLAN.md`: staging and commit strategy for release preparation.
- `SESSION_LOG.md`: recent working-session trace.
- `SESSION_TOKEN_REPORT.md`: session-level token and workflow reporting.
