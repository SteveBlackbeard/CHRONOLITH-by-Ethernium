# Chronolith

![Version](https://img.shields.io/badge/version-3.2.0-blue.svg)

Chronolith هو طبقة سلامة تشفيرية للمستندات التي تحمل نية المشروع عبر الجلسات المدعومة بالذكاء الاصطناعي. يقوم بتجزئة مستنداتك المرجعية في شجرة ميركل، ويسجّل خط أساس موقّعًا، ويتوقف توقفًا آمنًا عندما لا تعود الحالة الراهنة مطابقة له.

هو لا يضمن أن «يتذكر» الذكاء الاصطناعي. بل يمنحك سجلًا قابلًا للتحقق لما *كان عليه* السياق المتفق عليه، وتوقفًا حاسمًا عند انحرافه.

## الإصدارات

- **Lite**: تسليم محلي بالحد الأدنى وفحوصات حالة موقّعة.
- **Pro**: الحارس الكامل — خطوط أساس موقّعة بـ Ed25519، وسلسلة شفافية بالإضافة فقط، وبراهين اشتمال ميركل، والتشفير، وتدوير المفاتيح، والتثبيت على بيتكوين.
- **Omega**: رسم خرائط معرفية موجّه لـ RAG وتحليل الأثر.

## التثبيت

```bash
pip install chronolith
chronolith --help
```

حزم منفردة:

```bash
pip install chronolith-lite
pip install chronolith-pro
pip install chronolith-omega
```

## دليل قابل للتحقق

إحدى حالات هذا المشروع موثّقة زمنيًا في **كتلة بيتكوين 958484**. لست مضطرًا للوثوق بهذا المستودع ولا بمؤلفه — تحقّق بنفسك:

```bash
pip install "chronolith-pro[anchor]"
chronolith-pro verify-anchor --proof docs/evidence/ANCHOR_3647dd737ee8.json.ots
```

ما يثبته: أن تلك البيانات بعينها كانت موجودة قبل تعدين تلك الكتلة. ما لا يثبته: من كتبها، أو أنها صحيحة. هذه الحدود موثّقة وليست مخفية.

## حدود المنتج

Chronolith هو بيئة تشغيل Python ونواة الحوكمة. أما Seneschal فهو أداة منفصلة وقابلة للاستخراج لاقتصاد الرموز ومنح الصلاحيات الموقّعة.
