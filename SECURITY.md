# Security Policy: Ethernium Continuity Legacy 🏛️🛡️🦾

## 🏛️ Governance Philosophy
Ethernium Continuity Legacy is an industrial-grade synchronization framework. Our commitment is to the integrity of the "Logical Lineage" and the protection of the cognitive DNA of your software. Security is treated as a foundational nucleotide.

## 🛰️ Reporting Vulnerabilities
We take security seriously. If you discover an entropy risk, a memory core vulnerability, or a DNA synthesis flaw, please report it via one of the following channels:

- **Ethernium Forge (Issue Tracker)**: Open a [Private Vulnerability Report](https://github.com/SteveBlackbeard/CONTINUITY-LEGACY-by-Ethernium/security/advisories/new) (if available) or an issue with the label `[SECURITY]`.
- **Direct Linkage**: contact the Ethernium Sentinels.

### What NOT to Report Publicly:
-   Exploits that compromise `STATE.json` signatures.
-   Merkle Tree corruption vectors.
-   Authentication bypasses in OMEGA edition.

## 🔍 Validation vs. Integrity
Continuity is a **validation** tool. It is designed to detect drift, not necessarily to prevent malicious intrusion at the OS level. We recommend combining Continuity with encrypted filesystems and Git-signed commits for maximum sovereignty.

## 🧠 Supported Versions
We only provide security patches for the **Stable v2.1.x+ Branch**. All previous experimental branches (v1.0 - v2.0) are considered "Legacy Entropy" and are not maintained.
